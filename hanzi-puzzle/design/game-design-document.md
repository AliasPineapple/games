## 1. 产品定位

### 一句话描述

堆叠的偏旁方块铺满屏幕，点击能和目标偏旁组成汉字的字根，拼出目标数量的汉字即可通关——考验识字能力，也锻炼反应与专注力。

### 与现有产品的关系

```
汉字找一找        汉字拼图消消乐
  训练：识别          训练：构字原理
  问：哪些字含"氵"？  问：氵能和哪些字根组成汉字？
  方向：认识->辨别    方向：拆解->重组
```

两款游戏形成**学习闭环**：先会认（找一找），再会造（拼图消消乐），可在同一入口互相导流。

### 目标用户

- 核心：6-12 岁小学生（一到六年级，汉字构字学习的黄金期）
- 扩展：幼儿园大班（独体字叠字）、初中生（形声字、生僻字）
- 场景：课后碎片时间（3-8 分钟/局）、语文课前预习/复习

## 2. 核心玩法

### 2.1 玩法概述（参考"羊了个羊"）

游戏开始时，大量偏旁方块一次性飘落铺满屏幕，方块之间**可以相互重叠遮挡**（上层方块遮住下层方块的一部分），形成堆叠感。

屏幕顶部固定显示**目标偏旁**（如「氵」），玩家的任务是在堆叠的方块中，**点击能与目标偏旁组成汉字的字根**，达到目标数量即通关。

```
屏幕布局示意：

+------------------------------------------+
|  目标偏旁：[氵]  已找到：3/5  剩余：01:30 |
+------------------------------------------+
|                                          |
|    [青]  [木]  [天]                      |
|       [心] [可] [火]                     |
|   [日]   [方]    [土]                    |
|      [目]   [米] [月]                    |
|  [讠]    [胡]  [手]  [石]               |
|       ...（方块堆叠遮挡）                 |
|                                          |
+------------------------------------------+
|  结果展示区（横排格子）：[清][沐][___][___][___] |
+------------------------------------------+
```

### 2.2 基础循环（Core Loop）

```
游戏开始
    |
偏旁方块一次性飘落，堆叠铺满屏幕
    |
玩家点击某个方块
    |
    +---> 该字根 + 目标偏旁 = 合法汉字？
    |           |
    |         [是]  --> 正确动画（方块上浮消失）
    |                --> 底部格子出现合成的汉字
    |                --> 已找到数量 +1
    |                --> 检查是否达到目标数量
    |                         |
    |                      [达到] --> 通关动画 --> 下一关
    |                      [未达到] --> 继续等待点击
    |         [否]  --> 错误动画（方块红色闪烁归位）
    |                --> 扣除时间（-5秒）
    |
时间耗尽且未达到目标数量 --> 失败界面 --> 重试/退出
```

### 2.3 点选规则

**正确点击**：点击的字根能与目标偏旁组成一个真实合法的汉字。

示例（目标偏旁：氵）：
- 点击「青」-> 氵+青 = **清** (正确)
- 点击「木」-> 氵+木 = **沐** (正确)
- 点击「可」-> 氵+可 = **河** (正确)
- 点击「胡」-> 氵+胡 = **湖** (正确)
- 点击「方」-> 氵+方 = 不成字 (错误)
- 点击「天」-> 氵+天 = 不成字 (错误)

**遮挡规则**：只要方块有任意一点点露出来，就可以点击——哪怕只露出一个角。只有被其他方块**完全覆盖**（没有任何像素露出）才不可点击。这样玩家需要仔细观察堆叠层，寻找藏在下面的目标字根。

**说明**：每局屏幕上正确方块数量 >= 目标数量，保证关卡一定可以通关。

### 2.4 得分与评级

通关后根据以下指标给出星级评价：

| 评级 | 条件 |
|------|------|
| 3星 | 零错误 + 剩余时间 > 50% |
| 2星 | 错误 <= 2次 OR 剩余时间 > 20% |
| 1星 | 通关即可（无论多少错误）|
| 失败 | 时间耗尽，未达到目标数量 |

错误次数不影响通关，只影响星级和剩余时间（每次错误扣5秒）。

## 3. 关卡与难度设计

### 3.1 难度分级

| 难度 | 对应年级 | 目标偏旁类型 | 方块总数 | 目标数量 | 时限 |
|------|---------|------------|---------|---------|------|
| 幼儿园 | 幼儿园大班 | 叠字（木+木=林）| 12块 | 3个 | 90秒 |
| 小学低年级 | 1-2年级 | 基础偏旁（氵木口土日）| 16块 | 5个 | 75秒 |
| 小学高年级 | 3-4年级 | 形旁+声旁（氵讠忄火金）| 20块 | 5个 | 60秒 |
| 初中 | 5-6年级+初中 | 形声字+会意字 | 25块 | 6个 | 50秒 |
| 挑战模式 | 初中以上 | 生僻字多部件 | 30块 | 8个 | 45秒 |

**干扰项设计原则**：
- 干扰方块的字根**看起来像**能组成目标偏旁的字，但实际不合法（增加辨别难度）
- 幼儿园难度干扰项少（仅3-4个干扰块），挑战模式干扰项多且迷惑性强

### 3.2 解锁机制

- 每个难度阶段有 10 关，每关目标偏旁不同
- 通关（1星以上）解锁下一关
- 连续3星解锁隐藏关卡（生僻字挑战）
- 完成一个阶段全部10关，解锁下一难度阶段

## 5. 动效与反馈设计

### 5.1 正确点击动效

1. 方块高亮闪光（0.1秒）
2. 方块向上飘起，缩小消失（0.3秒，ease-out）
3. 目标偏旁方块与消失的方块汇聚到结果格子（0.4秒轨迹动画）
4. 结果格子中汉字出现，放大弹跳（0.2秒 scale 1.3 -> 1.0）
5. 格子短暂发光后恢复正常

### 5.2 错误点击动效

1. 方块变红，左右抖动（0.3秒 shake 动画）
2. 屏幕顶部计时进度条闪红，短暂缩短（视觉化扣时）
3. 方块恢复原样，仍在原位

### 5.3 通关动效

1. 所有结果格子同时发光
2. 从下到上的粒子上升效果（用 Canvas 实现，不依赖图片）
3. 星级评分动画（1颗/2颗/3颗星依次出现）
4. 「继续」按钮出现

### 5.4 失败动效

1. 计时条归零，变为灰色
2. 屏幕出现半透明蒙层
3. 显示「时间到！」文字 + 已找到数量
4. 「重试」「退出」按钮

## 6. 新手引导设计

**触发条件**：第一次进入第 1-1 关时自动触发，此后不再触发。

**引导步骤**：

```
步骤1：介绍目标
  -> 高亮顶部目标偏旁区域，文字提示：
     "这是今天的目标偏旁：氵（水字旁）"
     "找出能和氵组成汉字的字根，点击它！"
     [知道了] 按钮

步骤2：高亮正确方块
  -> 屏幕变暗，唯独「青」字块高亮发光
  -> 文字提示："氵 + 青 = 清，点击它！"
  -> 等待玩家点击「青」（只有这一块可以点）

步骤3：展示合成效果
  -> 正常播放正确动画，结果格子出现「清」
  -> 文字提示："太棒了！氵+青=清"
     "继续找，还需要 4 个！"
     [开始吧！] 按钮

步骤4：引导结束，正式进入游戏
  -> 计时开始
```

**引导期间**：计时器不启动，玩家无法点击非高亮方块。

## 7. 音效方案

**实现方式**：全部用 Web Audio API 程序生成，无需外部音频文件，单 HTML 文件即可完整运行。

| 音效 | 触发时机 | 合成方式 |
|------|---------|---------|
| 正确音 | 点击正确字根 | 短促上升音调（440Hz->880Hz，0.15秒）|
| 错误音 | 点击错误字根 | 短促下降噪声（200Hz->100Hz，0.2秒）|
| 通关音 | 达到目标数量 | 三音节上升旋律（do-mi-sol，各0.15秒）|
| 失败音 | 时间耗尽 | 下降长音（300Hz->100Hz，0.5秒）|
| 方块飘落音 | 游戏开始方块下落时 | 轻柔白噪声，极短（0.05秒×多次）|
| 计时警告音 | 剩余时间 < 15秒 | 每秒一次短促高频提示音（880Hz，0.1秒）|

**Web Audio API 合成示意（代码层面）**：

```javascript
function playCorrectSound() {
  const ctx = new (window.AudioContext || window.webkitAudioContext)();
  const osc = ctx.createOscillator();
  const gain = ctx.createGain();
  osc.connect(gain);
  gain.connect(ctx.destination);
  osc.frequency.setValueAtTime(440, ctx.currentTime);
  osc.frequency.exponentialRampToValueAtTime(880, ctx.currentTime + 0.15);
  gain.gain.setValueAtTime(0.3, ctx.currentTime);
  gain.gain.exponentialRampToValueAtTime(0.001, ctx.currentTime + 0.2);
  osc.start(ctx.currentTime);
  osc.stop(ctx.currentTime + 0.2);
}
```

## 8. 技术方案

### 8.1 技术栈

| 项目 | 选型 | 理由 |
|------|------|------|
| 渲染 | HTML5 + CSS（DOM方案）| 方块是静态布局，DOM 比 Canvas 更易实现点击检测和 CSS 动画 |
| 动效 | CSS Animation + JS | 正确/错误动画用 CSS keyframes，轨迹动画用 JS 计算位移 |
| 音效 | Web Audio API | 程序合成，无需外部文件，单 HTML 完整运行 |
| 数据存储 | localStorage | 进度/星级本地保存 |
| 字库 | JS 对象（内嵌） | 复用汉字找一找的字库思路，格式见下方 |
| 部署 | 单 HTML 文件 | 与汉字找一找保持一致 |

**选 DOM 而非 Canvas 的理由**：
- 方块是静态堆叠（无持续运动），不需要每帧重绘
- CSS 动画（shake、flyup、bounce）比手写 Canvas 动画更易维护
- 点击检测直接用 DOM 事件，无需手写碰撞检测
- 遮挡效果用 z-index 实现，简单直观

### 8.2 遮挡实现方案

方块使用**绝对定位**，随机分配 x/y 坐标和 z-index，模拟堆叠效果：

```javascript
// 方块数据结构
const blocks = [
  {
    id: 1,
    radical: "青",         // 字根字符
    x: 120,                // 像素坐标
    y: 280,
    zIndex: 5,             // 层级，越大越靠上
    isClickable: true,     // 是否可点击（未被完全遮挡）
    isCorrect: true,       // 是否为正确字根（游戏逻辑用，不显示给玩家）
  },
  // ...
];

// 遮挡判断：若某方块被上层方块覆盖面积 > 70%，则不可点击
function updateClickable(blocks) {
  blocks.forEach(block => {
    const coveredArea = calcCoveredArea(block, blocks.filter(b => b.zIndex > block.zIndex));
    block.isClickable = coveredArea < 0.7;
  });
}
```

### 8.3 偏旁字典数据结构

```javascript

const RADICAL_MAP = {
  "氵": {

    "木": "沐",
    "米": "汩",
    "方": null,   // 干扰项，氵+方不成字，值为 null
    "天": null,

  },

    "目": "相",
```

### 8.4 关键技术点

1. **方块初始飘落动画**：游戏开始时，所有方块从屏幕顶部飘落到目标位置，用 CSS transition 实现，错开延迟时间（每块相差 30-50ms）产生自然感。

2. **遮挡判断方案**：用矩形求交算法判断"完全遮挡"。一个方块被视为完全遮挡，当且仅当它的四个角都被更高层（z-index 更大）的方块覆盖。只要四个角中有任意一个角没被覆盖（露出一点），该方块就可以点击。实现时遍历所有高层方块的矩形范围，用布尔标记逐个角检测是否被覆盖。

3. **正确方块轨迹动画**：被点击的方块从当前位置飞向结果格子的对应位置，用 JS 计算起止坐标，CSS transform + transition 实现飞行轨迹。

## 10. 与汉字找一找的联动设计
