## 1. 产品定位

### 一句话描述

把偏旁部首拖拽拼合成完整汉字，消除方块得分——在玩消消乐的过程中，掌握汉字的构字规律。

### 与现有产品的关系

```
汉字找一找  ──>  汉字拼图消消乐
  训练：识别        训练：构字原理
  问：这些字里哪个含"氵"？  问：氵+青 能拼成什么字？
  方向：认识 -> 辨别    方向：拆解 -> 重组
```

两款游戏形成**学习闭环**：先会认（找一找），再会造（拼图消消乐），可在同一 APP 内互相导流。

### 目标用户

- 核心：6-12 岁小学生（一年级到六年级，正好是汉字构字学习的黄金期）
- 扩展：幼儿园大班（简单独体字 + 基础偏旁）、初中生（生僻字、形声字）
- 场景：课后碎片时间（5-10 分钟/局）、语文课前预习/复习

## 2. 核心玩法

### 2.1 基础循环（Core Loop）

```
偏旁方块从上方飘落
    |
玩家拖动两个偏旁拼合
    |
拼合成功 -> 显示完整汉字 -> 消除方块 -> 得分
    |
拼合失败 -> 方块变红闪烁 -> 归位 -> 扣时
    |
方块堆满屏幕 OR 时间耗尽 -> 游戏结束
```

### 2.2 拼合规则

每局选定一个「目标场景」，例如：

- 水字旁场景（氵）：玩家要用「氵」和其他偏旁拼出"河、湖、海、流、清、淡……"
- 木字旁场景（木）：拼出"树、林、桌、椅、柜、棵……"
- 心字旁场景（心/忄）：拼出"想、忘、忙、情、恨、悟……"

**拼合方式**：
1. 屏幕上同时存在「目标偏旁」方块 + 「配合偏旁/字根」方块
2. 玩家把「配合偏旁」拖到「目标偏旁」上方/下方/左侧/右侧（根据汉字结构）
3. 两块靠近到一定距离自动吸附，松手后判断是否合法

**合法判断**：
- 根据预定义的「偏旁组合字典」匹配
- 不需要玩家精确判断左右结构/上下结构，系统自动识别最匹配的汉字

### 2.3 得分机制

| 汉字类型 | 基础分 | 说明 |
|---------|--------|------|
| 简单合体字 | 10 分 | 如：日+月=明，木+木=林 |
| 形声字（常见） | 20 分 | 如：氵+青=清，木+目=相 |
| 形声字（较难） | 30 分 | 如：讠+青=请，心+青=情 |
| 生僻合体字 | 50 分 | 如：三个木=森，四个日=曌 |
| 连击加成 | x1.5 / x2 | 连续3个/5个不出错，乘以加成系数 |
| 速度加成 | +5 分 | 3秒内完成拼合额外加分 |

### 2.4 时间与节奏

- 单局时长：3 分钟（基础模式）/ 5 分钟（挑战模式）
- 方块下落速度：随游戏进行逐渐加快（约每30秒提速一级）
- 每成功消除一行方块，下落速度短暂降低（喘息时机）

## 4. 教育设计

### 4.1 知识融入方式

**隐性学习**（玩的过程中自然吸收）：
- 拼合成功后，汉字短暂放大显示，旁边有语义小图标（"清"旁边出现清水波纹动效）
- 连续拼出同一偏旁的字，出现「成就提示」："你已经认识了5个水字旁的字！"

**显性学习**（主动提示，不强制）：
- 游戏结束后「本局学到的字」回顾卡片（每个字+图片+拼音+简单例句）
- 可选开启「提示模式」：拖动偏旁时，显示该偏旁的语义（氵=水，点击查看更多）

### 4.2 课程标准对应

| 年级 | 语文课程标准要求 | 游戏对应关卡 |
|------|---------------|-------------|
| 一年级 | 认识独体字，了解简单偏旁 | 幼儿园+小学低年级1-5关 |
| 二年级 | 了解形声字构字规律 | 小学低年级6-10关 |
| 三-四年级 | 利用偏旁推测字义 | 小学高年级全部关卡 |
| 五-六年级 | 了解合体字结构（左右/上下/包围） | 初中难度关卡 |

### 4.3 避免「刷题感」

- 同一个字不会在短期内重复出现（记录最近50局的字，避免重复）
- 每局结束的学习卡片是「可跳过」的，不强制看
- 成就系统强调「你认识了多少字」而非「你答对了几道题」

## 5. 交互设计

### 5.1 核心交互

```
[触摸/鼠标拖拽] 偏旁方块
    -> 拖动中：方块略微放大 + 轻微阴影，有拖拽手感
    -> 靠近可拼合的方块时：两块出现「磁吸」效果，轻微抖动提示
    -> 松手判断：
        合法 -> 合并动画（两块旋转靠拢 + 爆炸粒子效果） + 得分浮字 + 短促音效
        不合法 -> 红色闪烁 + 震动反馈 + 方块弹回原位
```

### 5.2 界面布局

```
+----------------------------------+
|  场景名称   得分: 1250   [退出]   |  <- 顶部状态栏
|  [进度条]  目标: 氵 水字旁        |
+----------------------------------+
|                                  |
|    [  氵  ]  [  青  ]  [  木  ]  |  <- 方块区域（可拖动）
|                                  |
|    [  目  ]  [  心  ]  [  日  ]  |
|                                  |
|    [  方  ]  [  讠  ]  [  土  ]  |
|                                  |
+----------------------------------+
|  [提示] 剩余时间: 02:15  [暂停]  |  <- 底部操作栏
+----------------------------------+
```

### 5.3 动效要求

- 方块下落：缓入（ease-in）曲线，有轻微弹跳落地感
- 合并成功：粒子爆散 + 汉字短暂放大 + 光晕
- 连击提示：屏幕边缘出现光芒 + "2连击！" / "5连击！！"
- 关卡通关：烟花动效 + 星级评分动画

## 6. 技术方案

### 6.1 技术栈

| 项目 | 选型 | 理由 |
|------|------|------|
| 渲染引擎 | HTML5 Canvas（原生）或 Phaser 3 | 拖拽交互+粒子效果，Canvas 更流畅 |
| 触摸支持 | Pointer Events API | 统一鼠标和触屏 |
| 数据存储 | localStorage | 进度/得分本地保存，无需后端 |
| 字库 | JSON 文件 | 复用汉字找一找的字库格式 |
| 部署 | 单 HTML 文件 | 与汉字找一找保持一致 |

### 6.2 核心数据结构

```javascript
// 偏旁组合字典（核心数据）
const CHAR_COMBOS = {
  "氵": {
    "青": "清",  // 氵 + 青 = 清
    "方": "汪",  // 实际是氵+王=汪，需要精确校验
    "可": "河",
    "胡": "湖",
    // ...
  },
  "木": {
    "目": "相",
    "木": "林",  // 木 + 木 = 林
    "口": "杏",
    // ...
  },
  // ...
};

// 方块对象
class Block {
  constructor(radical, x, y) {
    this.radical = radical;  // 偏旁字符，如 "氵"
    this.x = x;
    this.y = y;
    this.width = 80;
    this.height = 80;
    this.isDragging = false;
    this.velocityY = 2;  // 下落速度
  }
}
```

### 6.3 关键技术挑战

1. **偏旁组合字典构建**：这是工作量最大的部分，需要整理常用合体字的偏旁拆分数据。建议先实现小学低年级的200个常用字，逐步扩充。

2. **拖拽吸附判断**：需要计算两个方块中心点距离，阈值约为方块宽度的 80%。

3. **方块布局防重叠**：方块下落时需要检测碰撞，避免堆叠在一起无法拖拽。

## 8. 美术风格

### 8.1 整体风格

- 参考：《汉字找一找》保持视觉一致性
- 风格：扁平化 + 轻拟物（方块有轻微立体感）
